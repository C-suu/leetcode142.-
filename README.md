### （1）带有详细注释的代码

```python
class Solution:
    def detectCycle(self, head: Optional[ListNode]) -> Optional[ListNode]:
        # 初始化慢指针 slow 和快指针 fast，起始位置均设定为链表的头节点 head
        slow = fast = head
        
        # 只要快指针不为空，且快指针的下一个节点也不为空，就持续执行循环（防止越界报错）
        while fast and fast.next is not None:
            # 慢指针每次向后移动一步
            slow = slow.next
            
            # 快指针每次向后移动两步
            fast = fast.next.next
            
            # 判断快慢指针是否在同一个节点相遇，若相遇则说明链表中存在环
            if fast is slow:  # 相遇
                
                # 开启寻找环入口的第二阶段：当 slow 指针和从头开始的 head 指针未重合时
                while slow is not head:  # 再走 a 步
                    # slow 指针每次向后移动一步
                    slow = slow.next
                    
                    # head 指针每次向后移动一步
                    head = head.next
                    
                # 当 head 和 slow 相遇时，该节点即为环的入口节点，返回此节点
                return slow
                
        # 如果外层 while 循环正常结束，说明快指针走到了链表末尾，链表无环，返回 None
        return None
```

### （2）代码逐行详细中文解释

  * `class Solution:`
    定义名为 `Solution` 的类，作为封装算法逻辑的容器。
  * `def detectCycle(self, head: Optional[ListNode]) -> Optional[ListNode]:`
    定义核心方法 `detectCycle`。接收单向链表的起始节点 `head` 作为参数。执行完成后，返回入环的第一个节点；如果链表无环，则返回 `None`。
  * `slow = fast = head`
    声明并初始化两个指针变量 `slow`（慢指针）和 `fast`（快指针）。在算法的最开始，这两个指针被统一放置在链表的出发点，即头节点 `head` 处。
  * `while fast and fast.next:`
    启动外层循环，用于检测环是否存在。循环的延续条件是：快指针当前所在的节点不是空节点（`None`），并且快指针的下一个节点也不是空节点。由于快指针每次需跨越两个节点，必须提前探测前方是否有足够的路径，避免发生空指针异常。若链表是一条直线无环，快指针必定会触碰边界从而打破此循环。
  * `slow = slow.next`
    在循环体内，控制慢指针 `slow` 向前推进。每次仅移动到相邻的下一个节点，即移动速度为 1 步。
  * `fast = fast.next.next`
    控制快指针 `fast` 向前推进。每次连续跨越两个节点，即移动速度为 2 步。
  * `if fast is slow:`
    进行碰撞检测。判断当前的快指针 `fast` 与慢指针 `slow` 是否指向了内存中的同一个节点对象。如果相等，说明跑得快的指针在环形跑道内“套圈”追上了跑得慢的指针，确凿证明了环的存在。
  * `while slow is not head:`
    进入算法的第二阶段（寻找环入口点）。这是一个基于严谨数学推导的循环：假设头节点到入环点的距离为 $a$，入环点到相遇点的距离为 $b$，相遇点再走距离 $c$ 就能回到入环点。根据快慢指针的路程关系（快指针路程是慢指针的 2 倍），可以推导出 $a = c + (n-1)(b+c)$。这意味着：**从链表头部出发的指针，与从相遇点出发的指针，如果每次都走一步，它们最终一定会精确地在“入环点”相遇。** 只要当前的 `slow` 指针和原头节点 `head` 指针没有重合，就继续循环。
  * `slow = slow.next`
    在第二阶段的循环中，原本的相遇点指针 `slow` 每次向后移动 1 步。
  * `head = head.next`
    在第二阶段的循环中，复用传入的 `head` 参数作为一个新的移动指针，从链表头部出发，每次也向后移动 1 步。
  * `return slow`
    当内层的 `while slow is not head:` 循环被打破时，说明 `head` 和 `slow` 成功指向了同一个节点。这个节点正是数学推导得出的环形起点。直接将其返回作为最终结果。
  * `return None`
    当整个外层大循环 `while fast and fast.next:` 因触及空节点而结束时，说明快指针走到了链表的尽头。由于存在尽头，链表绝不可能包含环，因此返回 `None`。

### （3）具体数值算例与变量变化表格

假设有一个包含 4 个节点的单向链表，数值和结构为：`3 -> 2 -> 0 -> -4`，其中末尾的节点 `-4` 的 `next` 指针重新指向了节点 `2`，构成了一个环。

节点与结构对应关系如下：

  * 链表头 `head` = 节点 `3`
  * 入环点 = 节点 `2`
  * 环内路径 = `2 -> 0 -> -4 -> 2 -> ...` 无限循环

**第一阶段：判断是否有环并寻找相遇点（快慢指针同跑）**

| 阶段 | 动作前 `slow` 指向 | 动作前 `fast` 指向 | 执行动作及指针新位置 | `if fast is slow` 碰撞判断 |
| :--- | :--- | :--- | :--- | :--- |
| **初始** | `3` (头节点) | `3` (头节点) | 未开始移动 | 不执行 |
| **第 1 次循环** | `3` | `3` | `slow` 走 1 步至 `2`<br>`fast` 走 2 步至 `0` | `2` $\neq$ `0`，不成立 |
| **第 2 次循环** | `2` | `0` | `slow` 走 1 步至 `0`<br>`fast` 走 2 步至 `2` | `0` $\neq$ `2`，不成立 |
| **第 3 次循环** | `0` | `2` | `slow` 走 1 步至 `-4`<br>`fast` 走 2 步至 `-4` | `-4` == `-4`，成立，证明有环！ |

此时，在节点 `-4` 处成功相遇，第一阶段结束。

**第二阶段：寻找入环节点（头指针与相遇点同跑）**

相遇点位于节点 `-4`。此时复用 `head` 指针（当前位于头节点 `3`），与 `slow` 指针（当前位于相遇点 `-4`）以相同的速度齐头并进。

| 阶段 | 动作前 `head` 指向 | 动作前 `slow` 指向 | `while slow is not head` 判断 | 执行动作及指针新位置 |
| :--- | :--- | :--- | :--- | :--- |
| **初始** | `3` (头节点) | `-4` (相遇点) | 成立 (`-4` $\neq$ `3`) | 两者同时移动一步 |
| **第 1 次循环** | `3` | `-4` | 成立 | `slow` 走 1 步至 `2`<br>`head` 走 1 步至 `2` |
| **第 2 次循环** | `2` | `2` | 不成立 (`2` == `2`) | 循环终止 |

**最终结果：**
内层循环在节点 `2` 处终止。程序执行 `return slow`，准确返回了入环的第一节点（节点 `2`）。
